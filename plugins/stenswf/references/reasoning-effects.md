# Reasoning effects (research-backed prompt primitives)

Last reviewed: 2026-04-25.
Re-evaluate when: a successor paper supersedes s1 (Stanford 2025) or
Renze & Guven 2024 — or when stenswf scope grows beyond personal
tooling and warrants an eval suite.

This file is the single source of truth for the three prompt primitives
inlined into stenswf skills. Skills inline the *templates* below
(customized per phase). The *citations* and *index* live only here, so
updates are a one-file edit.

---

## Templates

### step-back

Force the model to classify the problem before solving it. Separates
comprehension from execution; reduces pattern-matched hallucinations.

> Before producing the answer, step back and identify what *type* of
> problem this is: <axes that matter for this skill>. Name the class
> first; only then produce the artifact.

### pre-finalize-reflection

Force a second pass over the just-produced output before the model
commits to it. Targets premature termination of reasoning. Must ask a
*specific*, phase-grounded question — generic "did I miss anything?"
underperforms.

> Before finalizing, pause and step back: <phase-specific question
> grounded in this skill's failure modes>. If the answer changes the
> output, revise before continuing.

### constraint-repetition

Restate a binding constraint near the point where it must hold,
simulating bidirectional attention on a high-stakes invariant that sits
far from the action site.

> Reminder before proceeding: <verbatim restatement of the constraint
> stated earlier in this file>.

## Why these exist

- **s1 paper** (Stanford 2025): appending `Wait` to extend reasoning
  raised AIME 2024 scores from 50% → 56.7%, surpassing o1-preview.
  Mechanism: LLMs bias toward early termination; a pause signal
  overrides it.
- **Renze & Guven 2024** ("Self-reflection in LLM agents"): every type
  of self-reflection improved problem-solving with statistical
  significance (p < 0.001) across all tested LLMs. *Rich* reflections
  with specific hints outperformed sparse "retry"-style instructions.
- **Step-back prompting** (Google Research): classifying the problem
  before solving raised accuracy 72% → 89% on standard problems and
  61% → 85% on complex ones (~+17–24% absolute).
- **Prompt repetition**: restating constraints near the action site
  simulates bidirectional attention; reported 47 wins / 0 losses
  across 70 tests.

Underlying mechanism in all four: LLMs process tokens left-to-right and
commit early to a reasoning path. Forcing re-examination — via pause,
reflection, classification, or repetition — gives the model a second
pass over its own logic.

## Phrasing convention

Natural-language framing only. Do **not** use the literal `Wait` token
in instruction position — the s1 mechanism applies to mid-CoT
injection, not markdown instructions.

## Brevity interaction

`brevity` lists internal reasoning in its in-scope list, but no longer
actively rewrites it (the per-message self-check was removed). The
reflection passes generated by these templates may read terser than
ideal but should not be suppressed. If observed reflections collapse
to sparse "did I miss anything? no" outputs, that is the signal to
revisit the brevity/reflection trade-off — do not pre-emptively carve
out per skill.
